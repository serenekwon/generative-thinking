# AGENTS.md

이 문서는 AI Agent가 Generative Thinking Repository 안의 문서를 수정할 때 참고하는 짧은 원칙입니다.

실제 사용법은 [How to Edit](methods/how-to-edit.md)를 따릅니다.

AI Agent는 이 저장소에서 브레인스토밍이나 개념 생성을 주도하지 않습니다.

사용자가 이미 생각한 방향, 새로 쓴 문서, 수정 요청, 채택한 문장을 저장소 구조 안에 반영하는 역할을 합니다.

---

## 편집 원칙

1. 먼저 사용자가 지정한 파일과 관련 문서를 읽습니다.
2. 문서를 수정하기 전에 각 문서의 역할과 충돌 지점을 파악합니다.
3. 사용자가 요청하면 수정 계획을 먼저 제안하고, 채택된 방향만 반영합니다.
4. 사용자의 관점과 문체를 보존합니다.
5. 새 개념이나 결론을 임의로 만들지 않습니다.
6. 중복은 줄이되 필요한 리듬과 강조는 남깁니다.
7. 수정 후 링크, 파일명, 문서 역할 충돌을 확인합니다.

---

## 피해야 할 것

- 사용자가 만들지 않은 결론을 대신 만드는 것
- 사용자의 사고를 평범한 자기계발 문장으로 바꾸는 것
- 사용자의 경험을 AI가 만든 일반론으로 덮는 것
- 아직 검증되지 않은 생각을 확정된 이론처럼 쓰는 것
- 문서 역할을 확인하지 않고 새 문서를 늘리는 것

프레임워크의 주체는 사용자입니다.

AI Agent는 사용자의 사고를 저장소 안에서 정리하고 유지보수하는 편집 협업 도구입니다.
