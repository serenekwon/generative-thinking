# Core Terms

## 번역과 문체 원칙

이 문서는 Serene Human Framework를 다른 언어로 번역하거나 요약할 때 지켜야 할 핵심 용어와 문체 기준입니다.

각 언어에서 자연스러운 표현과 원문의 결을 보존하는 표현은 다를 수 있습니다.

이 저장소의 글은 짧은 문장, 반복을 통한 리듬, 선언적인 결론, 문단 사이의 여백을 중요하게 사용합니다.

번역할 때는 문장을 불필요하게 학술적으로 만들지 마십시오.

직역하지 않되 논리 구조는 바꾸지 마십시오.

반복과 여백을 무조건 제거하지 마십시오.

원문보다 더 과장하거나 철학적으로 포장하지 마십시오.

핵심 용어는 문서 전체에서 일관되게 번역하십시오.

특히 `Human Nature`와 `Self-Interest`처럼 짧고 압축된 글은 의미만큼 호흡이 중요합니다.

번역은 원문을 더 그럴듯하게 꾸미는 일이 아닙니다.

사용자의 목소리를 다른 언어에서도 가능한 한 살아 있게 유지하는 작업입니다.

---

## 권장 번역 절차

1. 먼저 `README.md`를 읽고 프로젝트의 목적과 기본 태도를 파악합니다.
2. `AGENTS.md`를 읽고 편집 원칙과 문체 기준을 확인합니다.
3. 이 문서를 읽고 핵심 용어의 번역을 고정합니다.
4. 전체를 한 번에 번역하지 말고 문서별로 번역합니다.
5. 마지막에 전체 문서의 용어, 호흡, 문체를 통합 검수합니다.

## 핵심 용어

| Korean | Preferred English | Notes |
| --- | --- | --- |
| 사고체계 | thinking framework | `thought system`보다 자연스럽고 프로젝트의 성격에 가깝습니다. |
| 인간행동모델 | human behavior model | 일반 개념으로 쓸 때 사용합니다. 프로젝트명은 `Serene Human Framework`를 유지합니다. |
| Human Energy Model | Human Energy Model | 고유 모델명으로 번역하지 않습니다. 필요할 때 `HEM`으로 줄일 수 있습니다. |
| 적합성 | fit | `compatibility`보다 짧고 실제 삶의 맞음, 환경과의 맞물림을 잘 담습니다. |
| 몰입 | flow / immersion | 행동과 에너지의 자연스러운 흐름은 `flow`, 깊이 빠져드는 상태는 `immersion`을 문맥에 따라 사용합니다. |
| 살아나다 | come alive | 이 프레임워크의 핵심 표현입니다. `be energized`보다 더 생동감 있는 기본 번역으로 둡니다. |
| 소진되다 | be depleted / burn out | 일시적인 에너지 고갈은 `be depleted`, 장기적 소진은 `burn out`을 사용합니다. |
| 세계관 | worldview | 문서 전체에서 일관되게 사용합니다. |
| 의미 | meaning | 과장해서 `purpose`로 바꾸지 않습니다. 문맥상 삶의 방향이면 `meaning`을 기본으로 둡니다. |
| 욕구 | desire / need | 하고 싶어지는 힘은 `desire`, 결핍이나 필요는 `need`를 사용합니다. |
| 기질 | temperament | 성격 평가가 아니라 반응 방식에 가깝게 다룹니다. |
| 보상 | reward | 행동 이후에 오는 이득이나 감각을 뜻합니다. |
| 시간 할인 | temporal discounting | 설명이 필요한 곳에서는 `discounting the future`로 풀어 쓸 수 있습니다. |
| 현재의 나 | present self | 시간과 보상 문서에서 사용합니다. |
| 미래의 나 | future self | 시간과 보상 문서에서 사용합니다. |
| 자기이해 | self-understanding | `self-knowledge`보다 과정의 느낌을 살립니다. |
| 관찰 | observation | 자기 판단이 아니라 있는 그대로 보는 태도를 담습니다. |
| 실험 | experiment | 삶 안에서 해보는 작은 시도를 뜻합니다. |
| 조건 | conditions | 사람을 탓하기 전에 봐야 할 환경, 대상, 시간, 의미의 조합입니다. |
| 공동연구원 | research partner | AI의 역할을 설명할 때 사용합니다. |
| 협업 도구 | collaborative tool | AI를 저자나 주체로 보지 않기 위한 표현입니다. |
| 유지보수 | maintenance | 문서와 사고체계를 계속 고치는 작업을 뜻합니다. |
| 리팩토링 | refactoring | 중복을 줄이고 구조를 명료하게 만드는 문서 작업에도 사용할 수 있습니다. |

## 번역 프롬프트 예시

`README.md`, `AGENTS.md`, 이 문서의 용어 기준을 먼저 읽고 번역하십시오.

직역하지 말되 논리 구조는 바꾸지 말고, 핵심 용어는 문서 전체에서 일관되게 유지하십시오.

짧은 문장, 반복, 여백, 선언적인 결론은 가능한 한 보존하십시오.

원문보다 과장하거나 학술적이고 철학적인 문체로 포장하지 마십시오.

## 피해야 할 번역

`적합성`을 자동으로 `compatibility`로 옮기지 마십시오.

이 문서에서 적합성은 사람과 대상, 환경, 시간, 의미가 맞물리는 감각입니다.

기본 번역은 `fit`입니다.

`살아나다`를 항상 `be motivated`로 옮기지 마십시오.

동기부여보다 더 넓은 감각입니다.

기본 번역은 `come alive`입니다.

`사고체계`를 지나치게 무겁게 `philosophical system`으로 옮기지 마십시오.

이 저장소는 철학 체계를 선언하기보다 생각을 정리하고 유지보수하는 프레임워크에 가깝습니다.

기본 번역은 `thinking framework`입니다.
